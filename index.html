🐙 Subir a GitHub Pages

Una vez en GitHub Pages, toda tu equipo accede desde el navegador — sin instalar nada.

1
Descargar el archivo Descarga cruce-agendas.html desde esta plataforma.
2
Crear repositorio en GitHub Ve a github.com/new y crea un repositorio público con el nombre que prefieras, por ejemplo:
cruce-agendas-vt
3
Subir el archivo Dentro del repositorio creado, haz clic en "Add file → Upload files", sube el HTML y renómbralo a index.html antes de subir (o renómbralo en GitHub después).
4
Activar GitHub Pages Ve a Settings → Pages en tu repositorio:
· Source: Deploy from a branch
· Branch: main → carpeta / (root)
· Clic en Save.
5
Tu URL pública quedará así:
https://[tu-usuario].github.io/cruce-agendas-vt/
Comparte esa URL con tu equipo. Cualquiera puede usarla sin instalar nada.
6
Actualizar cuando haya cambios Solo sube el nuevo index.html al mismo repositorio (drag & drop en GitHub). GitHub Pages se actualiza automáticamente en ~1 minuto.
✔ Nota sobre el historial: Los cruces guardados viven en el navegador de cada usuario (localStorage). No se sincronizan entre computadoras — cada persona tiene su propio historial local. Para compartir resultados usa el botón 📄 Reporte para Jefes que genera un HTML descargable.
